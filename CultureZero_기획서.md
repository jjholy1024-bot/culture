# CultureZero 기획서
### 2026 외교 공공데이터·AI 활용 경진대회

> 이 문서는 Claude Code로 실제 구현을 진행하기 위한 작업 기준 문서입니다.

---

## 1. 서비스 개요

**서비스명**: CultureZero
**한 줄 소개**: 국가를 검색하거나 지도에서 클릭하면, 외교부 공공데이터와 AI가 함께 만든 안전·문화 브리핑을 5분 안에 받아볼 수 있는 서비스

**타겟 사용자**: 해외 여행자, 유학생, 단기 파견 직장인

**핵심 가치**:
- 흩어진 안전·문화 정보를 한 곳에서 신뢰도 있게 제공
- 외교부 공식 데이터(사실 정보) + AI(맥락 정보)를 명확히 구분해서 보여줌으로써 정보의 출처를 투명하게 공개

---

## 2. 데이터 흐름

```
사용자가 국가 검색 또는 지도 핀 클릭
        ↓
미리 만들어둔 국가별 JSON 파일 fetch
        ↓
6개 카드 + 상단 배지로 브리핑 화면 구성
```

데이터는 실시간 API 호출이 아니라, 사전에 6개 외교부 API를 호출해 가공한 뒤 정적 JSON 파일로 만들어두고 프론트에서는 그 파일만 fetch하는 구조 (자세한 내용은 6번 항목).

---

## 3. API별 활용 내역

| # | API명 | 활용 범위 | 화면 내 위치 |
|---|---|---|---|
| 1 | 사건사고 예방정보 (AccidentService) | 응답값 전체 활용 | "사건사고 예방정보" 카드 |
| 2 | 여행경보제도 | 경보 단계(level)만 활용 | 결과 화면 상단 배지 |
| 3 | 국가별 국기 이미지 (CountryFlagService2) | 국가 단위 이미지만 (지역 이미지 미사용), 다운로드 URL로 받아 `/images` 폴더에 저장 후 사용 | 국기 원형 아이콘 |
| 4 | 국가별 공지사항 목록조회 (CountryNoticeService) | 외부 링크만 제공 (상세 내용은 미표시) | "외교부 공지사항 바로가기" 버튼 |
| 5 | 국가·지역별 현지연락처 (LocalContactService2) | 응답값 전체 활용 | "긴급 연락처" 카드 |
| 6 | 안전공지 | 선별적으로 활용 (사건사고 예방정보와 중복되지 않는 내용 위주로 보완) | "사건사고 예방정보" 카드 내 보충 정보 |

문화·예절·비즈니스 팁·현지 표현 4가지 항목은 외교부 API에 해당 데이터가 없으므로 **AI가 생성**하여 채움.

## 4. 화면 구성

### 화면 1 — 메인
- 검색창 (국가명 입력)
- 지도 뷰 (기본) — 핀 클릭 시 해당 국가 브리핑으로 이동, 여행경보 단계별로 핀 색상 차등 표시
- 인기 국가 칩 리스트 (지도/리스트 토글 가능)

### 화면 2 — 로딩
- 데이터 fetch → 카드 구성 단계별 진행 표시

### 화면 3 — 결과 (브리핑)
상단: 국기 이미지 + 국가명 + 여행경보 단계 배지

카드 6개 (2단 그리드):
- 🛡️ 사건사고 예방정보 *(외교부 데이터)*
- 🚦 여행경보 단계 상세 *(외교부 데이터)*
- 📞 긴급 연락처 *(외교부 데이터)*
- 🤝 문화·예절 *(AI 생성)*
- 💼 비즈니스 팁 *(AI 생성)*
- 💬 유용한 현지 표현 *(AI 생성)*

하단: 외교부 공지사항 링크 / 공유 / PDF 저장

## 6. 데이터 구조 설계 (정적 JSON 방식)

GlobalRecruit과 동일하게 **백엔드 서버 없이 정적 JSON 파일**로 처리. 6개 API를 미리 호출해 데이터를 가공한 뒤, 국가 단위로 통합된 JSON 파일을 레포에 저장해두면 프론트는 그 파일을 fetch만 하는 구조.

### 왜 국가별 통합 방식인가
API별로 파일을 따로 두면(`accident.json`, `alert.json`...) 국가 하나를 보여주기 위해 프론트에서 최대 5번 fetch해야 하고, 그만큼 로딩이 느려지고 코드도 복잡해짐. 국가 단위로 미리 합쳐두면 fetch 한 번으로 카드 6개를 모두 채울 수 있어 훨씬 효율적.

### 파일 구조

```
public/data/
├── countries.json              ← 전체 국가 목록 (가벼운 정보만, 메인 화면용)
└── countries/
    ├── JPN.json                ← 국가별 상세 정보 (상세 화면용)
    ├── USA.json
    ├── FRA.json
    └── ...
```

### ① countries.json — 메인 화면(지도·검색·인기국가)용

지도에 핀 찍고, 검색 자동완성하고, 인기 국가 칩 보여주는 데 필요한 최소 정보만 담음. 용량이 가벼워야 메인 화면 로딩이 빠름.

```json
[
  {
    "iso_code": "JPN",
    "country_kr": "일본",
    "country_en": "Japan",
    "lat": 36.2048,
    "lng": 138.2529,
    "flag_image": "images/JPN.png",
    "alert_level": "없음"
  }
]
```

### ② countries/JPN.json — 상세 브리핑 화면용

국가 클릭/검색 시 해당 파일 하나만 추가로 fetch해서 카드 6개를 채움.

```json
{
  "iso_code": "JPN",
  "country_kr": "일본",
  "country_en": "Japan",
  "flag_image": "images/JPN.png",
  "travel_alert": {
    "level": "없음"
  },
  "accidents": [
    { "title": "...", "content": "..." }
  ],
  "local_contacts": [
    { "office": "주일본대사관", "phone": "..." }
  ],
  "notice_url": "https://www.0404.go.kr/...",
  "culture_ai": {
    "etiquette": "...",
    "business_tip": "...",
    "phrases": ["...", "..."]
  }
}
```

`culture_ai` 필드는 데이터 수집 스크립트 단계에서 AI로 미리 생성해 JSON에 박아두는 방식으로 진행. (실시간 호출 방식은 채택하지 않음 — 로딩 속도와 API 비용 면에서 사전 생성이 유리)

### 이미지 저장 방식

국기 이미지는 데이터 수집 스크립트에서 다운로드 URL로 미리 받아 `public/images/국가코드.png` 형태로 레포에 저장해두고, 프론트는 `flag_image` 경로를 그대로 `<img src="...">` 에 사용 (요청마다 새로 다운로드하지 않음).

> 국기 이미지를 다운로드해 저장하는 스크립트는 GlobalRecruit의 `collect_data.py`와는 별도의 스크립트로 둠.

### 프론트 동작 흐름

```
페이지 진입 → countries.json 1회 fetch → 지도 핀 + 인기국가 칩 렌더링
국가 클릭/검색 → countries/[ISO코드].json 1회 fetch → 카드 6개 렌더링
```

## 7. 작업 전 확인이 필요한 부분

### (1) 각 API의 실제 응답(response) 필드명
현재 가진 자료는 요청 파라미터(request)까지만 확인된 상태. 실제 응답이 어떤 JSON 구조로 오는지(`response.body.items.item` 형태인지, 필드명이 영문인지 국문인지 등)는 API를 직접 호출해보고 확인할 예정.

### (2) "안전공지" API
다른 5개 API와 달리 정확한 엔드포인트명이 특정되지 않은 상태. 공공데이터포털에서 직접 검색해 정확한 서비스명과 호출 방식을 확인할 예정.

### (3) 사용자가 입력하는 국가명과 API 파라미터 매칭
API가 `isoCode`, `country_nm`(한글), `country_en_name`(영문) 등 여러 방식으로 국가를 받기 때문에, 검색창에 "일본"을 입력했을 때 이를 적절한 코드로 변환해 API에 넘길 매핑표가 필요함.
→ 전체 국가 리스트 + ISO 코드 매핑표를 별도로 작성해서 진행.
